# React Project Setup

## Learning Objectives
- Scaffold a React + TypeScript project using Vite or Create React App
- Understand the generated project structure and key configuration files
- Start the development server and verify the application runs

## Why This Matters

Setting up a project correctly from the start saves hours of debugging later. A well-scaffolded React + TypeScript project comes with the compiler configured, the dev server ready, and the folder structure organized. This is the first hands-on step in our weekly epic -- by the end of this reading, you will know exactly how to go from an empty directory to a running React application.

## The Concept

### Scaffolding Tools

Two popular tools scaffold React + TypeScript projects with zero manual configuration:

#### Option A: Vite (Recommended)

Vite is a modern build tool that provides near-instant dev server startup and fast hot module replacement.

```bash
npx create-vite@latest taskflow-app --template react-ts
cd taskflow-app
npm install
npm run dev
```

#### Option B: Create React App (CRA)

CRA was the original scaffolding tool for React. It is still functional, but Vite is generally preferred for new projects due to its speed.

```bash
npx create-react-app taskflow-app --template typescript
cd taskflow-app
npm start
```

### Project Structure (Vite + React + TypeScript)

After scaffolding with Vite, the project looks like this:

```
taskflow-app/
  node_modules/          # Installed dependencies (do not edit or commit)
  public/
    vite.svg             # Static assets served as-is
  src/
    App.tsx              # Root component
    App.css              # Styles for the root component
    main.tsx             # Application entry point
    index.css            # Global styles
    vite-env.d.ts        # Vite type declarations
  index.html             # The single HTML page (SPA entry)
  package.json           # Project manifest
  tsconfig.json          # TypeScript compiler configuration
  tsconfig.node.json     # TypeScript config for Vite's Node.js context
  vite.config.ts         # Vite configuration
```

### Key Files Explained

#### `index.html`

The single HTML page that hosts the entire SPA. It contains a `<div id="root">` where React mounts:

```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>TaskFlow</title>
  </head>
  <body>
    <div id="root"></div>
    <script type="module" src="/src/main.tsx"></script>
  </body>
</html>
```

#### `src/main.tsx` (Entry Point)

This file initializes React and renders the root component into the DOM:

```typescript
import React from "react";
import ReactDOM from "react-dom/client";
import App from "./App";
import "./index.css";

ReactDOM.createRoot(document.getElementById("root")!).render(
  <React.StrictMode>
    <App />
  </React.StrictMode>
);
```

Key details:
- `ReactDOM.createRoot` creates a React root attached to the `#root` div.
- `React.StrictMode` enables additional development-time checks and warnings.
- The `!` after `getElementById("root")` is a TypeScript non-null assertion -- it tells the compiler this element will always exist.

#### `src/App.tsx` (Root Component)

The starting component. Replace its contents with your own:

```typescript
function App() {
  return (
    <div>
      <h1>TaskFlow</h1>
      <p>Your React + TypeScript application is running.</p>
    </div>
  );
}

export default App;
```

#### `tsconfig.json`

The TypeScript configuration generated by Vite is already optimized for React development:

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "useDefineForClassFields": true,
    "lib": ["ES2020", "DOM", "DOM.Iterable"],
    "module": "ESNext",
    "skipLibCheck": true,
    "moduleResolution": "bundler",
    "allowImportingTsExtensions": true,
    "isolatedModules": true,
    "moduleDetection": "force",
    "noEmit": true,
    "jsx": "react-jsx",
    "strict": true,
    "noUnusedLocals": true,
    "noUnusedParameters": true,
    "noFallthroughCasesInSwitch": true
  },
  "include": ["src"]
}
```

Notable settings:
- `"jsx": "react-jsx"` -- enables the automatic JSX runtime (React 17+).
- `"strict": true` -- all strict type-checking options are on.
- `"noEmit": true` -- Vite handles the actual compilation; `tsc` is used only for type checking.

#### `vite.config.ts`

Minimal Vite configuration for React:

```typescript
import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";

export default defineConfig({
  plugins: [react()],
});
```

### Running the Development Server

```bash
npm run dev
```

This starts the Vite dev server, typically at `http://localhost:5173`. The server provides:

- **Hot Module Replacement (HMR):** Changes to your code appear instantly in the browser without a full reload.
- **TypeScript type checking:** Errors are displayed in the terminal and browser overlay.
- **Fast startup:** Vite serves modules on demand via native ESM, so startup is nearly instant regardless of project size.

### Building for Production

```bash
npm run build
```

This compiles TypeScript, bundles all modules, minifies the output, and writes the result to `dist/`. The output is static HTML, CSS, and JavaScript that can be deployed to any web server.

### Recommended Folder Structure

As the project grows, organize your source code by feature or function:

```
src/
  components/       # Reusable UI components
    Header.tsx
    TaskCard.tsx
  pages/            # Page-level components (one per route)
    HomePage.tsx
    DashboardPage.tsx
  hooks/            # Custom hooks
  types/            # Shared TypeScript interfaces and types
    task.ts
  utils/            # Utility functions
  styles/           # Global and shared styles
  App.tsx
  main.tsx
```

This structure keeps the codebase navigable as it grows.

## Code Example

End-to-end: creating a project and verifying it works:

```bash
# 1. Scaffold the project:
npx create-vite@latest taskflow-app --template react-ts

# 2. Install dependencies:
cd taskflow-app
npm install

# 3. Start the dev server:
npm run dev

# 4. Open http://localhost:5173 in your browser.

# 5. Edit src/App.tsx -- the browser updates automatically.
```

## Summary

- Use Vite with the `react-ts` template (or CRA with `--template typescript`) to scaffold a React + TypeScript project.
- The key files are `index.html` (SPA shell), `main.tsx` (entry point), `App.tsx` (root component), `tsconfig.json` (TypeScript config), and `vite.config.ts` (bundler config).
- `npm run dev` starts the dev server with hot module replacement.
- Organize source code by feature or function as the project grows.
